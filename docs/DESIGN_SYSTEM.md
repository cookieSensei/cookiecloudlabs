# CookieSensei Design System

Version: 1.0

---

# Purpose

The CookieSensei Design System defines how every website, curriculum page,
presentation, document, and application should look, feel, and communicate.

Our goal is consistency.

Whether someone visits our homepage, reads our curriculum, attends a workshop,
or uses CookieSensei Labs, the experience should feel like it belongs to the
same educational institution.

---

# Design Philosophy

Technology can already feel overwhelming.

Our design should never add to that complexity.

Every interface should be:

• Calm
• Minimal
• Professional
• Approachable
• Educational

We do not design to impress.

We design to help people learn.

---

# Core Principles

## 1. Simplicity over decoration

Avoid unnecessary animations, effects, gradients, and visual clutter.

Whitespace is content.

Every element should have a purpose.

---

## 2. One idea per section

Each section should answer one important question.

Examples:

Why CookieSensei Exists

↓

How Learning Happens

↓

Learning Without Barriers

↓

Our Mission

Readers should never feel overwhelmed.

---

## 3. Clear language

Avoid unnecessary technical jargon.

Instead of

Pedagogy

say

How We Teach

Instead of

Experiential Learning

say

Learning by Building

Our writing should be understandable by beginners while still respected by professionals.

---

## 4. Confidence without hype

We don't exaggerate.

We don't promise unrealistic outcomes.

We present ideas clearly and honestly.

Confidence comes from clarity.

---

## 5. Show, then explain

Whenever possible:

Show an idea

↓

Demonstrate it

↓

Explain it

Avoid long paragraphs before examples.

---

# Typography

## Hero Titles

```text
text-[clamp(3rem,7vw,5.5rem)]
```

Used only for page heroes.

---

## Section Titles

```text
text-[clamp(2rem,5vw,3.75rem)]
```

Used for major sections.

---

## Body Text

```text
text-lg md:text-xl
```

Readable on every screen size.

---

## Section Labels

```text
uppercase
tracking-[0.3em]
text-blue-400
font-semibold
```

Examples:

OUR MISSION

OUR APPROACH

THE LEARNING JOURNEY

---

# Layout Principles

Every page follows a simple rhythm.

Section Label

↓

Large Heading

↓

Supporting Paragraph

↓

Visual Content

↓

One memorable closing sentence

Whitespace separates ideas.

Avoid large walls of text.

---

# Writing Style

Every paragraph should be conversational.

Avoid marketing language.

Avoid unnecessary adjectives.

Use short sentences.

Use active voice.

Example:

Programming doesn't give you ideas.

It gives you the ability to recognize ideas you couldn't see before.

Simple.

Memorable.

Honest.

---

# Color Philosophy

Dark backgrounds improve focus.

Blue represents curiosity, learning, and trust.

Accent colors should be used sparingly.

Important actions should be obvious without becoming distracting.

---

# Component Philosophy

Pages should compose components.

Components should contain content.

Avoid large page files.

Preferred structure:

app/

page.tsx

↓

components/

Hero.tsx

Mission.tsx

LearningJourney.tsx

etc.

---

# Educational Philosophy

Every learner should feel:

"I can do this."

Technology should feel approachable.

Learning should begin with curiosity,
not software installation.

Programming is not the goal.

Programming is a tool for solving meaningful problems.

---

# CookieSensei Voice

CookieSensei is:

Calm.

Patient.

Curious.

Thoughtful.

Professional.

Never arrogant.

Never intimidating.

Never overly sales-focused.

We teach.

We don't sell.

---

# Design Checklist

Before publishing any new page, ask:

✓ Does every section have a clear purpose?

✓ Is the language understandable to beginners?

✓ Is there enough whitespace?

✓ Is the page visually consistent?

✓ Does every element help learning?

✓ Does this page feel like CookieSensei?

If the answer is "no", simplify.
